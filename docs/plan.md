# Terraform Template — Plan

Multi-cloud Terraform module templates for AWS, Azure, and GCP.

## Repo structure

```
terraform-template/
├── .github/workflows/ci.yml       # fmt + validate + test on PR
├── .pre-commit-config.yaml        # terraform_fmt, terraform_validate, terraform_docs
├── CONTRIBUTING.md                # module conventions
├── README.md
├── docs/plan.md
├── aws/                           # AWS modules
├── azure/                         # Azure modules
│   └── aks/
│       ├── main.tf
│       ├── variables.tf
│       ├── outputs.tf
│       ├── versions.tf
│       ├── README.md              # generated by terraform-docs
│       └── tests/basic.tftest.hcl
└── gcp/                           # GCP modules
```

Modules consumed via:

```hcl
source = "git::https://github.com/simonangel-fong/terraform-template.git//azure/aks?ref=v1.0.0"
```

## Conventions

- `versions.tf` pins Terraform `>= 1.6` and provider versions
- `variables.tf` — required vars on top, optional below; all vars typed and described
- `outputs.tf` — sensitive values marked `sensitive = true`
- `README.md` auto-generated by `terraform-docs`
- `tests/*.tftest.hcl` — at least one plan-mode test per module
- Modules do **not** configure providers — caller's responsibility

## Phase 1 — Bootstrap (AKS module)

**Scope**

- `azurerm_kubernetes_cluster` with system-assigned managed identity
- Default node pool (vm_size, node_count, auto-scaling min/max)
- Optional Log Analytics workspace + OMS agent (toggle via `enable_log_analytics`)

**Out of scope for v1** — add when needed:

- Additional node pools, private cluster, ACR integration, AAD/RBAC

**Inputs**

- Required: `name`, `location`, `resource_group_name`, `kubernetes_version`
- Optional: `default_node_pool`, `network_profile`, `tags`, `enable_log_analytics`

**Outputs**

- `cluster_id`, `cluster_name`, `kube_config_raw` (sensitive), `kubelet_identity`, `oidc_issuer_url`

**Assumptions**

- Caller creates the resource group
- Caller configures the `azurerm` provider
- AzureRM provider pinned to `~> 4.0`

## Phase 2 — Cloud parity

- AWS EKS module (mirror AKS scope)
- GCP GKE module (mirror AKS scope)

## Phase 3 — Networking

- Azure VNet
- AWS VPC
- GCP VPC

## Phase 4 — Additional services

Storage, identity, and observability modules as needed.

## CI

`.github/workflows/ci.yml` runs on PR:

- `terraform fmt -check -recursive`
- Per module: `terraform init -backend=false`, `terraform validate`, `terraform test`
