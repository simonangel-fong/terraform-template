# Contributing

## Module conventions

Every module follows the same layout:

```
<cloud>/<service>/
├── main.tf
├── variables.tf
├── outputs.tf
├── versions.tf
├── README.md
└── tests/
    └── basic.tftest.hcl
```

### Rules

- **`versions.tf`** pins `required_version = ">= 1.6"` and all providers.
- **`variables.tf`** — required variables on top, optional below. Every variable has `description` and `type`.
- **`outputs.tf`** — mark sensitive values with `sensitive = true`.
- **`README.md`** — usage snippet and requirements section. Inputs/outputs tables are generated by `terraform-docs` between the `BEGIN_TF_DOCS` / `END_TF_DOCS` markers — do not hand-edit.
- **`tests/*.tftest.hcl`** — at least one `command = plan` test per module so CI runs without cloud credentials.
- **Modules do not configure providers.** No `provider "azurerm" {}` block inside a module — that's the caller's job.
- **Modules do not create resource groups, projects, or accounts.** Callers pass these in.

## Versioning

Tag releases as `vMAJOR.MINOR.PATCH`. Modules are consumed by tag:

```hcl
source = "git::https://github.com/simonangel-fong/terraform-template.git//<cloud>/<service>?ref=v1.0.0"
```

## Local checks before opening a PR

```sh
terraform fmt -recursive
terraform -chdir=<cloud>/<service> init -backend=false
terraform -chdir=<cloud>/<service> validate
terraform -chdir=<cloud>/<service> test
```
